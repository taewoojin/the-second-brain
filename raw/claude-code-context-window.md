# Claude Code 컨텍스트 윈도우 — 작동 원리와 구성

**출처**: https://code.claude.com/docs/en/context-window
**날짜**: 2026-04-20

Claude Code의 컨텍스트 윈도우는 Claude가 현재 세션에서 알고 있는 모든 것을 담는 공간이다. 사용자의 지시사항, Claude가 읽은 파일, Claude 자신의 응답, 그리고 터미널에는 나타나지 않는 숨겨진 내용까지 모두 포함된다. 최대 200,000 토큰의 한계가 있으며, 그 안에서 무엇이 어느 순서로 쌓이는지를 이해하는 것이 효율적인 사용의 핵심이다.

---

## 기본 개념: 가시성(Visibility)

컨텍스트에 존재한다고 해서 터미널에 보이는 것은 아니다. 모든 컨텍스트 항목은 세 가지 가시성 수준 중 하나를 갖는다.

| 가시성 | 의미 |
|--------|------|
| **완전 표시** (full) | 실제 내용이 터미널에 표시됨 |
| **한 줄 표시** (brief) | 터미널에 짧은 요약만 표시됨 (예: "Read auth.ts") |
| **숨김** (hidden) | 터미널에 전혀 나타나지 않음 |

---

## 이벤트 종류(Kind)

컨텍스트에 쌓이는 항목들은 그 출처에 따라 구분된다.

| 종류 | 의미 |
|------|------|
| **auto** | 자동으로 로드되는 항목 (사용자가 아무것도 하지 않아도 로드됨) |
| **you** | 사용자가 직접 입력한 내용 |
| **claude** | Claude의 작업 (파일 읽기, 분석, 수정 등) |
| **hook** | 자동으로 실행되는 훅(hook) |
| **compact** | /compact 명령으로 생성된 요약 |
| **subagent** | 서브에이전트의 별도 컨텍스트 윈도우에서의 작업 |

## 1. 세션 시작 전: 자동 로드 항목

`claude` 명령을 실행하면 사용자가 아무것도 입력하기 전에 아래 항목들이 자동으로 컨텍스트에 로드된다.

### 시스템 프롬프트 (System prompt)
- **가시성**: 숨김
- Claude의 핵심 행동 방식, 도구 사용법, 응답 형식에 대한 지시사항
- 항상 가장 먼저 로드됨
- 사용자는 절대 볼 수 없음

### 자동 메모리 (Auto memory / MEMORY.md)
- **가시성**: 숨김
- Claude가 이전 세션에서 스스로 기록한 메모 (빌드 명령어, 발견한 패턴, 피해야 할 실수 등)
- 앞에서부터 200줄 또는 25KB 중 더 작은 크기까지만 로드됨
- 관련 문서: https://code.claude.com/docs/en/memory#auto-memory

### 환경 정보 (Environment info)
- **가시성**: 숨김
- 작업 디렉토리, 플랫폼, 쉘, OS 버전, Git 여부 등
- Git 브랜치, 상태, 최근 커밋은 시스템 프롬프트 맨 끝에 별도 블록으로 로드됨

### MCP 도구 (MCP tools — deferred)
- **가시성**: 숨김
- 사용 가능한 MCP 도구의 **이름만** 먼저 로드됨 (전체 스키마는 필요할 때 개별 로드)
- Claude는 작업 중 도구가 필요하면 tool search를 통해 그때 스키마를 가져옴
- 설정 옵션:
  - `ENABLE_TOOL_SEARCH=auto`: 모든 스키마가 컨텍스트의 10% 이내에 맞으면 시작 시 전체 로드
  - `ENABLE_TOOL_SEARCH=false`: 시작 시 모든 스키마를 로드 (deferred 없이)
- 관련 문서: https://code.claude.com/docs/en/mcp#scale-with-mcp-tool-search

### 스킬 설명 (Skill descriptions)
- **가시성**: 숨김
- Claude가 스스로 호출할 수 있는 스킬들의 한 줄 설명 목록
- 전체 스킬 내용은 실제로 사용될 때만 로드됨
- **주의 1**: `disable-model-invocation: true`가 설정된 스킬은 이 목록에 없음 — 사용자가 직접 `/스킬명`으로 호출하는 순간까지 컨텍스트에 아예 존재하지 않음
- **주의 2**: `/compact` 이후 재주입되지 않음 — 실제로 호출했던 스킬만 보존됨
- 관련 문서: https://code.claude.com/docs/en/skills

### 전역 CLAUDE.md (`~/.claude/CLAUDE.md`)
- **가시성**: 숨김
- 모든 프로젝트에 적용되는 전역 설정
- 세션 시작 시 프로젝트 지시사항과 함께 로드됨
- 관련 문서: https://code.claude.com/docs/en/memory#choose-where-to-put-claude-md-files

### 프로젝트 CLAUDE.md (Project CLAUDE.md)
- **가시성**: 숨김
- 프로젝트 컨벤션, 빌드 명령어, 아키텍처 메모 등
- 가장 중요한 파일. 프로젝트 루트에 위치하여 팀 전체에 동일한 지시사항 공유
- **권장**: 200줄 이하로 유지. 참조용 내용은 스킬이나 경로 한정 규칙(path-scoped rules)으로 분리
- 관련 문서: https://code.claude.com/docs/en/memory

> **핵심 교훈**: 사용자가 아무것도 입력하기 전에 이미 많은 것이 로드된다. CLAUDE.md, 메모리, 스킬 목록, MCP 도구가 첫 번째 프롬프트 이전에 이미 컨텍스트에 있다.

## 2. 사용자 프롬프트

- **가시성**: 완전 표시
- 사용자가 입력한 텍스트가 컨텍스트에 추가됨
- 이미 로드된 항목들(CLAUDE.md, 메모리 등)에 비하면 상대적으로 매우 작은 크기

> **핵심 교훈**: 대부분의 컨텍스트는 프로젝트 지식(CLAUDE.md, 메모리, 규칙 등)이지, 사용자의 말 자체가 아니다.

## 3. Claude의 작업

### 파일 읽기 (File reads)
- **가시성**: 한 줄 표시 (터미널에 "Read auth.ts" 등으로만 표시)
- 파일 내용은 Claude만 봄 — 사용자에게는 보이지 않음
- **파일 읽기가 컨텍스트 사용의 주요 원인**이 됨
- **팁**: 프롬프트를 구체적으로 작성할수록 Claude가 읽는 파일이 줄어듦 (예: "버그 수정해줘" 대신 "auth.ts의 버그를 수정해줘")
- 리서치가 많은 작업은 서브에이전트를 활용하면 메인 컨텍스트를 아낄 수 있음

### 경로 한정 규칙 (Path-scoped rules)
- **가시성**: 한 줄 표시 (터미널에 "Loaded .claude/rules/api-conventions.md" 등으로 표시)
- `.claude/rules/` 안의 규칙 파일에 `paths:` 패턴이 있으면, 해당 경로의 파일을 읽을 때 자동으로 로드됨
- 규칙 내용은 Claude만 봄, 사용자에게는 보이지 않음
- 예: `paths: src/api/**`가 설정된 규칙은 Claude가 해당 디렉토리 파일을 읽을 때 자동 로드됨
- 예: `paths: *.test.ts`가 설정된 규칙은 테스트 파일을 읽을 때 자동 로드됨
- 관련 문서: https://code.claude.com/docs/en/memory#path-specific-rules

### grep 및 검색
- **가시성**: 한 줄 표시
- 코드베이스 검색 결과가 컨텍스트에 추가됨 — 사용자에게는 명령어 실행 사실만 보임

### Claude의 분석과 파일 수정
- **가시성**: 완전 표시
- Claude가 작성한 분석 텍스트, 파일 수정 diff가 터미널에 그대로 표시됨

### 테스트 실행 결과
- **가시성**: 한 줄 표시
- 터미널에는 명령어와 통과 수만 표시됨
- 전체 출력은 Claude의 컨텍스트에만 존재

> **핵심 교훈**: Claude가 읽는 파일 하나하나가 컨텍스트를 소모한다. 경로 한정 규칙은 매칭되는 파일을 읽을 때 자동으로 함께 로드된다.

## 4. 훅 (Hooks)

- **가시성**: 숨김 (터미널에 나타나지 않음)
- `settings.json`에 정의된 PostToolUse 훅이 파일 수정 등 도구 이벤트 후 자동 실행됨
- 예: 파일을 수정할 때마다 prettier 훅이 자동으로 실행됨

훅의 출력이 Claude 컨텍스트에 들어가는 방식:

| 훅 출력 방식 | Claude에게 전달 여부 |
|-------------|---------------------|
| `additionalContext` 필드가 있는 JSON 출력 | Claude의 컨텍스트에 전달됨 |
| 종료 코드 2 + stderr | Claude에게 오류로 표면화됨 (단, 도구가 이미 실행된 후라 막을 수 없음) |
| 종료 코드 0의 일반 stdout | 디버그 로그에만 기록, 컨텍스트에 들어가지 않음 |

- **팁**: 훅 출력을 간결하게 유지할 것 — 잘림(truncation) 없이 그대로 컨텍스트에 들어감
- 관련 문서: https://code.claude.com/docs/en/hooks-guide

> **핵심 교훈**: 훅은 도구 이벤트마다 자동으로 실행된다. Claude에게 정보를 전달하려면 반드시 `additionalContext` JSON 형식을 사용해야 한다.

## 5. 추가 프롬프트 (Follow-up)

- 후속 프롬프트는 **동일한 컨텍스트에 계속 쌓임**
- 이전의 모든 내용(파일 읽기 결과, Claude의 분석, 수정 내용 등)이 그대로 남아 있음

> **핵심 교훈**: 후속 질문은 새로운 컨텍스트가 아니라 기존 컨텍스트에 추가된다. 세션이 길어질수록 컨텍스트가 누적된다.

## 6. 서브에이전트 (Subagent)

서브에이전트는 **별도의 컨텍스트 윈도우**에서 작동하는 독립된 Claude 인스턴스다. 파일을 많이 읽는 리서치 작업을 메인 컨텍스트 밖에서 처리하는 데 유용하다.

### 메인 세션에서 보이는 것
- 서브에이전트 생성 이벤트가 컨텍스트에 추가됨 (한 줄 표시)
- 서브에이전트가 읽은 파일들은 **메인 컨텍스트에 들어오지 않음**
- 작업 완료 후 **서브에이전트의 최종 텍스트 응답 + 작은 메타데이터(토큰 수, 소요 시간)만** 메인 컨텍스트로 돌아옴

### 서브에이전트의 별도 컨텍스트 구성

1. **자체 시스템 프롬프트**: 메인보다 짧음. 메인 세션의 자동 메모리는 포함되지 않음.
   - 예외: 커스텀 에이전트에 `memory:` 설정이 있으면 그 에이전트만의 별도 MEMORY.md를 로드함
2. **프로젝트 CLAUDE.md**: 동일한 파일이지만 서브에이전트의 컨텍스트에서 소비됨
   - 예외: 내장된 Explore, Plan 에이전트는 더 작은 컨텍스트를 위해 CLAUDE.md 로드를 건너뜀
3. **MCP 도구 + 스킬**: 부모와 동일한 MCP 서버 및 스킬에 접근 가능
   - 제외: 플랜 모드 제어 도구, 백그라운드 태스크 도구, 기본적으로 Agent 도구 자체 (재귀 방지)
4. **부모가 작성한 태스크 프롬프트**: 사용자 프롬프트 대신 Claude가 작성한 작업 지시를 받음
5. **파일 읽기 등 자체 작업**: 전부 서브에이전트의 컨텍스트에서만 소비됨, 메인에 영향 없음

> **핵심**: 서브에이전트가 대량의 파일을 읽어도 메인 컨텍스트에는 최종 요약만 돌아온다. 이것이 컨텍스트 절약의 핵심 메커니즘이다.

관련 문서: https://code.claude.com/docs/en/sub-agents

## 7. 특수 입력 방식

### 뱅 명령어 (`!` 접두사)
- **가시성**: 완전 표시
- 사용자가 `!git status` 처럼 `!`를 붙여 쉘 명령어를 직접 실행할 때 사용
- 명령어와 그 출력이 **사용자 메시지의 일부로** 컨텍스트에 추가됨
- Claude가 직접 명령어를 실행하지 않고도 그 결과를 컨텍스트에 포함시킬 수 있음
- 관련 문서: https://code.claude.com/docs/en/interactive-mode#bash-mode-with-prefix

### `disable-model-invocation: true` 스킬
- **가시성**: 한 줄 표시 (사용자가 호출하는 시점에만)
- 스킬 설명 목록에 없으므로 시작 시 컨텍스트 비용이 0
- 사용자가 `/스킬명`으로 호출하는 순간 전체 스킬 내용이 로드됨
- **권장 사용 사례**: 커밋, 배포, 메시지 전송 등 부작용(side effect)이 있는 스킬
- 관련 문서: https://code.claude.com/docs/en/skills#control-who-invokes-a-skill

## 8. /compact — 컨텍스트 압축

컨텍스트가 많이 쌓여 성능에 영향을 줄 때, 또는 장시간 작업 전에 사용하는 명령어.

- **가시성**: 한 줄 표시 ("Conversation compacted" 메시지만 보임)
- 대화 내용을 구조화된 요약으로 교체함
- 요약화 과정 자체는 터미널에 표시되지 않음

### 요약에 보존되는 내용
- 사용자의 요청과 의도
- 핵심 기술 개념
- 검토하거나 수정한 파일과 중요 코드 스니펫
- 오류와 수정 방법
- 미완료 작업
- 현재 진행 중인 작업

### 요약 후 손실되는 내용
- 도구 출력 원본 전체 (예: 파일 내용 그대로, 테스트 전체 출력 등)
- 중간 추론 과정

> Claude는 작업을 계속 참조할 수 있지만, 이전에 읽었던 코드의 정확한 내용은 더 이상 갖고 있지 않다.

관련 문서: https://code.claude.com/docs/en/how-claude-code-works#the-context-window

## 9. /compact 이후 생존 메커니즘

긴 세션이 압축될 때, 각 메커니즘이 어떻게 되는지:

| 메커니즘 | /compact 이후 상태 |
|----------|-------------------|
| 시스템 프롬프트 및 출력 스타일 | 변화 없음 — 메시지 히스토리가 아닌 별도 영역 |
| 프로젝트 루트 CLAUDE.md 및 스코프 없는 규칙 | 디스크에서 자동 재주입됨 |
| 자동 메모리 (Auto memory) | 디스크에서 자동 재주입됨 |
| `paths:` 프론트매터가 있는 경로 한정 규칙 | **소실** — 다음에 매칭 파일을 읽을 때 다시 로드됨 |
| 서브디렉토리의 중첩 CLAUDE.md | **소실** — 해당 서브디렉토리의 파일을 읽을 때 다시 로드됨 |
| 호출한 스킬 본문 | 재주입됨 (스킬당 최대 5,000 토큰, 전체 최대 25,000 토큰 제한; 초과 시 오래된 것부터 삭제) |
| 스킬 설명 목록 | **재주입되지 않음** — 실제로 호출했던 스킬만 보존됨 |
| 훅 | 해당 없음 — 훅은 코드로 실행됨, 컨텍스트와 무관 |

### 경로 한정 규칙 및 중첩 CLAUDE.md 주의사항
- 이들은 매칭 파일 읽기 시 **메시지 히스토리**에 로드됨
- 따라서 /compact 시 다른 대화 내용과 함께 요약되어 사라짐
- 컴팩션 이후에도 규칙을 유지하려면: `paths:` 프론트매터를 제거하거나, 프로젝트 루트 CLAUDE.md로 이동할 것

### 스킬 본문 주의사항
- 대용량 스킬은 스킬당 토큰 제한에 맞게 파일 앞부분부터 보존하고 잘릴 수 있음
- 따라서 **가장 중요한 지시사항을 SKILL.md 상단에** 배치할 것
- 오래된 스킬이 먼저 삭제되므로, 전체 토큰 예산 초과 시 오래 전에 호출한 스킬은 사라질 수 있음

> **핵심 교훈**: 컴팩션 후에는 시스템 설정, CLAUDE.md, 메모리가 자동으로 재로드된다. 스킬 설명 목록은 예외 — 실제 호출했던 스킬만 보존된다.

## 10. 내 세션 실시간 확인

시각화의 수치는 예시값이다. 실제 세션의 컨텍스트 상황을 확인하는 방법:

- **`/context`**: 카테고리별 실시간 컨텍스트 사용량 분석 + 최적화 제안
- **`/memory`**: 시작 시 로드된 CLAUDE.md 파일과 자동 메모리 파일 확인

## 11. 컨텍스트 절약 팁 (원문 팁 모음)

| 상황 | 팁 |
|------|-----|
| 파일 읽기 | 프롬프트를 구체적으로 작성할수록 Claude가 읽는 파일이 줄어듦 |
| 리서치 작업 | 서브에이전트를 활용하면 대량 파일 읽기가 메인 컨텍스트를 소모하지 않음 |
| 부작용 스킬 | `disable-model-invocation: true`로 설정하면 호출 전까지 컨텍스트 비용이 0 |
| CLAUDE.md 관리 | 200줄 이하로 유지; 참조용 내용은 스킬이나 경로 한정 규칙으로 분리 |
| 훅 출력 | `additionalContext` JSON을 간결하게 작성할 것 — 잘림 없이 그대로 컨텍스트에 들어감 |
| 긴 세션 | `/compact`를 사용해 컨텍스트를 정리; 새 장시간 작업 전에도 활용 |
