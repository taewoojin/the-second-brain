## 1단계: 버전 확인

Claude Code 버전을 확인하세요.

```bash
claude --version
```

v2.1.32 미만이면 업데이트가 필요합니다.

---

## 2단계: Agent Teams 활성화

Agent Teams는 기본적으로 꺼져 있으므로 먼저 활성화해야 합니다.

### 방법 1: 설정 파일에서 (권장)

`~/.claude/settings.json` 파일을 열고 다음을 추가합니다:

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

파일이 없으면 새로 만들어도 됩니다.

### 방법 2: 환경 변수로 (한 번만)

터미널에서 다음 명령으로 실행하면 그 세션에서만 활성화됩니다:

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude
```

## 3단계: 첫 팀 만들기

Claude Code를 시작하고 팀이 필요한 상황을 설명하세요.

```text
예시 1 (PR 검토):
PR #42를 여러 각도에서 리뷰해줄래?
- 한 명은 보안 문제 찾기
- 한 명은 성능 영향 확인
- 한 명은 테스트 커버리지 확인
```

```text
예시 2 (버그 조사):
사용자들이 한 메시지 후 앱이 종료된다고 해.
팀을 만들어서 여러 가설을 동시에 테스트해줄래?
```

```text
예시 3 (문서 작성):
README를 여러 섹션으로 나눠서 동시에 작성해줄래?
```

Claude가 팀을 만들 건지 물어볼 수도 있고, 자동으로 만들 수도 있습니다.

---

## 4단계: 디스플레이 모드 선택

팀원들을 어떻게 볼 것인지 선택할 수 있습니다.

**기본값 (in-process)**  
모든 팀원이 터미널 하나에서 돌아갑니다. Shift+Down을 눌러서 팀원들을 순회할 수 있고, 모든 터미널에서 작동합니다.

**Split pane (선택사항)**  
각 팀원이 독립된 창에 나타나서 모두의 진행을 동시에 볼 수 있습니다. tmux 또는 iTerm2이 필요합니다.

Split pane을 쓰고 싶으면:

```json
~/.claude.json에 다음 추가:
{
  "teammateMode": "tmux"
}
```

또는 한 번만 쓰려면:

```bash
claude --teammate-mode tmux
```

## 5단계: 팀 종료하기

작업이 끝났으면:

```text
팀을 정리해줄래?
```

Claude가 팀을 깔끔하게 정리해줍니다.

---

## 자주 묻는 질문

**Q. 팀원들이 보이지 않는데?**  
A. Shift+Down을 눌러보세요. in-process 모드에서는 순회해야 봅니다.

**Q. 팀원이 일을 너무 느리게 진행해요.**  
A. 직접 Shift+Down으로 이동해서 지시를 줄 수 있습니다.

**Q. 팀 리더가 먼저 종료되면?**  
A. 팀원들도 자동으로 종료됩니다. 진행 중인 작업이 있으면 먼저 정리하세요.
