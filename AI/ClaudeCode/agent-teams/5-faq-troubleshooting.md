# Q&A와 트러블슈팅

## 자주 묻는 질문

### 기본 개념 관련

**Q: Agent Teams와 그냥 Subagents의 차이가 뭐야?**

A: Subagents는 당신이 일꾼을 부르고 결과만 받는 방식입니다. 일꾼들끼리는 대화하지 않습니다. 빠르지만 병렬로 못 하는 일들이 있습니다.

Agent Teams는 여러 팀원이 팀을 이루고 서로 대화하면서 조정합니다. 느리지만 더 유연합니다.

**Q: 팀원들이 같은 파일을 수정하면 어떻게 되나?**

A: 한쪽이 다른 쪽을 덮어씁니다. 반드시 다른 파일로 분리해야 합니다.

**Q: 팀원과 리더가 동시에 같은 파일을 수정할 수 있나?**

A: 기술적으로 가능하지만 매우 위험하므로 피해야 합니다.

### 팀 만들고 운영하기

**Q: 팀을 만들었는데 팀원이 안 보여요.**

A: 
- In-process 모드: **Shift+Down**을 눌러서 순회해보세요. 팀원이 이미 있을 거예요.
- Split pane 모드: tmux가 설치되었는지 확인하세요 (`which tmux`).

**Q: 팀 리더가 실수로 종료되면?**

A: 팀원들도 자동으로 종료됩니다. 진행 중인 작업이 있으면 먼저 마무리해야 합니다.

**Q: 팀을 중간에 멈출 수 있나?**

A: 팀 리더가 정리를 요청하면 종료됩니다. 진행 중인 작업이 있으면 먼저 끝날 때까지 기다립니다.

**Q: 같은 팀 세션을 나중에 다시 시작할 수 있나?**

A: 현재로서는 어렵습니다. `/resume`는 in-process 팀원을 복구하지 못하므로 새 팀을 만드는 것이 낫습니다.

### 팀원 관리

**Q: 팀원이 일을 너무 느리게 진행해요.**

A: Shift+Down으로 그 팀원에게 이동한 후 구체적인 지시를 주거나, 리더가 직접 개입해서 방향을 수정한다

**Q: 팀원이 일을 완료했다고 표시했는데 실제로는 덜했어요.**

A:
```text
"잠깐, 이 부분 확인해봐. 아직 테스트를 추가하지 않았네."
```

팀원이 다시 그 작업을 진행합니다.

**Q: 팀원이 일을 거부했어요.**

A: 
```text
"왜 이건 안 하고 싶어?"
```

팀원의 이유를 들어보세요. 합리적이면 다른 접근을 시도하거나, 다른 팀원을 시켜보세요.

### 비용과 성능

**Q: Agent Teams 사용하면 토큰이 얼마나 많이 들어?**

A: 팀원 수 × 일반 Claude Code 비용입니다.
- 3명 팀: 3배
- 5명 팀: 5배

따라서 혼자 하는 것보다 3배 이상 빨라야 가치가 있습니다.

**Q: 팀원이 많을수록 좋지 않나?**

A: 아니요. 5명 이상이면 의사소통 비용이 얻는 이득을 넘을 수 있습니다.

## 일반적인 문제와 해결법

### 문제: 팀원이 일을 시작하지 않음

**증상**: 팀원이 작업 목록을 보지만 아무것도 하지 않음  
**원인**: 작업이 불명확하거나, 다른 작업에 의존하거나, 팀원이 방향을 잃음

**해결법**: 리더가 특정 작업부터 시작하도록 지시하고, 구체적으로 설명한다

### 문제: 팀원들끼리 대화가 없음

**증상**: 각 팀원이 독립적으로만 일하고 협력하지 않음  
**원인**: 팀원들이 서로의 진행 상황을 모르거나, 명시적으로 소통하도록 지시받지 않음

**해결법**: 리더가 팀원들에게 서로의 진행 상황을 확인하도록 지시한다

### 문제: 한 팀원이 다른 팀원의 일을 방해함

**증상**: 팀원 A가 팀원 B의 파일을 수정해서 충돌  
**원인**: 처음부터 파일 분리를 명확하게 하지 않음

**해결법**: 리더가 각 팀원에게 담당할 폴더를 명확하게 지정한다

### 문제: 팀원이 권한 때문에 막혔어요

**증상**: 팀원이 도구 사용 권한이 필요함 (파일 쓰기, bash 실행 등)

**원인**: 리더의 권한 설정이 제한적  
**해결법**: 리더가 권한을 더 주거나, 리더가 직접 그 작업을 진행한다

### 문제: Split pane이 안 돼요

**증상**: tmux를 찾을 수 없다는 에러  
**원인**: tmux가 설치되지 않았거나 PATH에 없음

**해결법**

macOS (Homebrew):
```bash
brew install tmux
```

Linux:
```bash
sudo apt install tmux
```

또는 in-process 모드 사용:
```bash
claude --teammate-mode in-process
```

## 팀이 일을 끝내지 않은 상황

### 진행이 느림

**진단**: 팀원이 뭘 하고 있는지 확인하고, 작업이 너무 큰지 살펴본다  
**해결**: 작업을 작은 것부터 시작하도록 하고, 구체적인 지시를 제공한다

### 팀원이 막혔음

**증상**
```
팀원: "이 부분 어떻게 할까요?"
```

**해결**: 리더가 단계별로 구체적인 지시를 제공한다

### 접근이 잘못됨

**증상**
```
팀원이 잘못된 방향으로 1시간을 작업했음
```

**방지법**: 중요한 작업은 "계획 승인"을 요구하고, 자주 체크인한다  
**복구**: 리더가 팀원에게 다른 접근을 제안하고 방향을 수정한다

## 문제 해결 체크리스트

문제가 생겼으면 이 순서로 진단한다:

1. 팀원의 상태 확인 (Shift+Down으로 이동)
2. 작업 목록 확인 (Ctrl+T)
3. 명확한 지시 제공 (다시 설명)
4. 부분 작업으로 나누기 (작은 단위로)
5. 리더가 개입 (막히면 도움)
6. 팀원 교체 (잘 안 되면 다른 팀원)
