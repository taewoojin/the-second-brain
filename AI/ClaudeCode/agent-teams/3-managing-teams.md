# 팀 운영하기

## 한눈에 보기

팀이 생기면 세 가지만 알면 됩니다:
1. 팀원과 메시지로 소통하기
2. 작업 목록으로 일 분배하기
3. 팀 정리하기

---

## 팀원과 소통하기

### 팀 리더가 팀원에게 지시하기

자연스럽게 말하면 됩니다:

```text
연구팀 팀원에게: PR #42의 보안 부분을 더 깊게 봐줄래?
```

Claude가 지시를 해당 팀원에게 전달합니다.

### 팀원이 묻고 싶을 때

팀원이 막히거나 확인이 필요하면 자동으로 리더에게 메시지를 보냅니다.

### In-process 모드에서 팀원 찾기

**Shift+Down**을 반복해서 누르면 팀원들을 순회합니다:

```
리더 (당신) → 팀원 1 → 팀원 2 → 팀원 3 → ... → 리더로 돌아옴
```

찾은 팀원에게 직접 메시지를 보낼 수 있습니다.

### Split pane 모드에서

마우스로 팀원의 창을 클릭하면 그 팀원과 직접 대화할 수 있습니다.

## 작업 목록으로 일 분배하기

### 팀 리더가 일을 분배하는 방식

**자동 분배 (권장)**
```text
이 코드를 세 부분으로 나눠서 병렬 리팩토링해줄래?
```

Claude가 작업 목록을 만들고, 팀원들에게 할당하고, 진행 상황을 추적합니다.

**명시적 분배**
```text
팀원 1에게 인증 모듈 리팩토링 할당해줄래
팀원 2에게 권한 확인 로직 할당해줄래
```

### 팀원들이 자동으로 일 선택하기

팀원이 현재 일을 끝내면 자동으로 다음 일을 찾아서 시작합니다.

---

## 작업 상태 확인하기

### In-process 모드에서

**Ctrl+T**를 누르면 작업 목록이 보입니다:

```
[ ] 작업 1 (대기 중)
[→] 작업 2 (팀원 A가 진행 중)
[✓] 작업 3 (완료)
```

### 리더에게 상태 물어보기

```text
지금 각 팀원이 뭘 하고 있어?
```

---

## 팀 리더가 결정 내리기

### 복잡한 작업에 계획 승인 요구하기

리더는 팀원들이 변경하기 전에 계획을 먼저 보고 승인할 수 있습니다:

```text
보안 리뷰어에게 auth 모듈 리팩토링을 부탁하는데,
반드시 계획을 먼저 보여주고 내가 승인한 후에 시작해줄래
```

팀원이 계획을 제출하면:
- 리더가 검토
- 괜찮으면 "승인", 문제 있으면 피드백과 함께 "거부"
- 팀원이 피드백을 반영해서 다시 제출

### 팀원 종료하기

팀원이 일을 마치지 않았는데 더 이상 필요 없으면:

```text
분석팀 팀원을 종료해줄래
```

리더는 종료 요청을 보내고, 팀원이 승인하면 종료됩니다.

---

## 팀 정리하기

모든 팀원이 일을 끝내면:

```text
팀을 정리해줄래
```

리더가 자동으로 진행 중인 작업을 확인하고, 모두 끝날 때까지 기다린 후, 팀 리소스를 정리합니다.

항상 리더가 정리 명령을 내려야 합니다. 팀원이 정리하면 문제가 생길 수 있습니다.

---

## 팀원과 효율적으로 일하기

### 충분한 정보 주기

팀원은 리더의 과거 대화를 모르므로 필요한 배경을 직접 설명해야 합니다:

```text
(좋은 예)
팀원에게: "src/auth/ 폴더의 보안 문제를 찾아줄래? 
우리는 JWT를 httpOnly 쿠키에 저장하고 있어."

(부족한 예)
팀원에게: "보안 검토해줄래?"
```

### 팀원이 막혔을 때

```text
팀원이 말함: "어디부터 봐야 할지 모르겠어"

리더가 대답:
"먼저 token validation 부분을 살펴봐. 
특히 만료된 토큰 처리가 제대로 되는지 확인해줄래"
```

## 자주 묻는 질문

**Q. 팀원들이 같은 파일을 수정하면?**  
A. 충돌이 생깁니다. 처음부터 파일을 분리해서 할당하세요.

**Q. 팀원이 일을 완료했다고 표시했는데 실은 덜했어요.**  
A. 직접 확인 후 "아직 해야 할 부분이 있어"라고 피드백 주세요.

**Q. 팀원끼리는 대화할 수 있나요?**  
A. 네, 필요하면 자동으로 합니다. 리더가 강제하지 않아도 됩니다.
