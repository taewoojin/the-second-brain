# Agent Teams가 뭔가요?

## 한눈에 보기

Agent Teams는 여러 Claude Code 세션을 팀으로 묶어서 병렬로 작업하는 기능입니다. 팀 리더가 일을 분담하고 조정하면, 팀원들이 각자 독립적으로 진행합니다.

---

## Agent Teams가 왜 필요한가요?

혼자 일하면 한 번에 한 가지만 생각할 수 있습니다. 팀으로 일하면 달라집니다.

- **여러 각도에서 동시 검토**: 같은 문제를 다양한 관점에서 본다
- **병렬 진행**: 독립적인 작업들을 동시에 끝낸다
- **상호 검증**: 누락된 부분을 다른 사람이 발견한다

**예를 들어:**
- PR 리뷰: 한 명이 보안, 한 명이 성능, 한 명이 테스트를 동시에 확인
- 버그 조사: 여러 팀원이 서로 다른 가설을 동시에 검증
- 새 기능 개발: 프론트엔드, 백엔드, 테스트를 각각 담당

---

## Agent Teams와 Subagents의 차이

비슷해 보이지만 목적이 다릅니다.

**Subagents (기존 방식)**  
당신이 도우미들을 부르고 결과를 받는 방식입니다. 도우미들끼리는 대화하지 않습니다. 빠르지만, 평행으로 진행하기 어려운 작업이 있습니다. 특정 조사나 분석 결과가 필요할 때 적합합니다.

**Agent Teams (새 방식)**  
여러 팀원이 독립적으로 일하고 필요할 때 서로 대화합니다. 팀 리더가 조정하지만 팀원들은 자율적으로 판단합니다. 처리능력(토큰)을 더 사용합니다. 팀원들의 의견과 상호작용이 중요할 때 효율적입니다.

---

## 실제 동작 흐름

```
당신 (팀 리더)
  ↓
"팀을 만들고 이런 일을 해줄래?"
  ↓
Claude Code가 팀 생성 + 팀원 배치
  ↓
팀원 1        팀원 2        팀원 3
(작업 A)      (작업 B)      (작업 C)
  ↓             ↓             ↓
[공유 작업 목록으로 조정]
  ↓             ↓             ↓
필요하면 팀원들끼리 메시지 주고받음
  ↓
작업 완료 후 리더에게 보고
```

---

## Agent Teams를 쓰면 안 되는 경우

같은 파일을 여러 명이 수정해야 하면 충돌이 생깁니다. 순차적 작업(A 완료 후 B 시작)도 팀의 이점을 살리기 어렵습니다. 간단한 일은 혼자가 빠르고, 토큰 비용도 많이 들어갑니다.
