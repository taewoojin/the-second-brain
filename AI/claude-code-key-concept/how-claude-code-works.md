# 목표: 
에이전트 루프, 내장 도구, Claude Code가 프로젝트와 상호작용하는 방식을 이해하고, 효과적으로 사용하기 위한 팁을 학습한다.

---

# 에이전트 루프
Claude에게 작업을 주면 세 가지 단계를 거침
1. 컨텍스트 추적
2. 작업 수행
3. 결과 검증

클로드는 코드를 이해하기 위해 파일 검색, 편집, 확인을 하기 위해 도구를 사용한다.

![에이전틱 루프](assets/agentic-loop.svg)

**루프는 작업 유형에 따라 유연하게 조정된다**
- 코드베이스 질문: 컨텍스트 수집만으로 충분
- 버그 수정: 3단계 반복 사이클
- 리팩토링: 광범위한 검증 포함
각 단계에서 이전 단계의 학습 결과를 기반으로 다음 행동을 결정하며, 수십 개의 작업을 연결하고 도중에 방향을 수정한다.
 
 **사용자도 루프의 일부:**
언제든지 개입해서 방향 전환, 추가 컨텍스트 제공, 다른 접근 방식으로 요청 가능. Claude는 자율적으로 작동하지만 사용자 입력에 반응한다.

에이전틱 루프는 두가지 구성 요소에 의해 구동된다.
1. 추론하는 모델
2. 실행하는 도구
Claude Code는 두 요소를 감싸는 에이전틱 하네스(harness) 역할을 하며 도구, 컨텍스트 관리 및 실행 환경을 제공해 언어 모델을 실용적인 코딩 에이전트로 변환한다.

# 모델
Claude Code는 Claude 모델을 사용하여 아래와 같은 일을 할 수 있다.
- **코드 이해**: 모든 언어의 코드를 읽고, 컴포넌트 간 연결 관계 파악
- **작업 추론**: 목표 달성을 위해 무엇을 변경해야 할지 판단
- **복잡한 작업 처리**: 단계 분해 → 실행 → 결과 기반으로 조정 반복

**모델 장단점**:
Sonnet: 일반적인 코딩 작업
Opus: 복잡한 아키텍처 설계를 위한 깊은 추론 제공

# 도구
도구는 Claude Code를 에이전트로 만드는 핵심.
도구가 없으면 Claude는 텍스트로만 응답할 수 있다.

## 내장 도구 5가지 범주 
| **범주** | **주요 기능** |
| --- | --- |
| 파일 작업 | 읽기, 편집, 생성, 재구성 |
| 검색     | 패턴/정규식으로 파일·코드 탐색 |
| 실행     | 셸 명령, 서버, 테스트, git |
| 웹      | 검색, 문서 fetch, 에러 조회 |
| 코드 인텔리전스 | 타입 오류 확인, 정의 이동, 참조 찾기 (플러그인 필요) |

## 실제 동작 예시 — "실패한 테스트 수정해"
```
1. 테스트 실행
2. 에러 읽기
3. 관련 파일 검색
4. 코드 이해
5. 파일 수정
6. 테스트 재실행 ✅
```

각 단계의 결과가 다음 단계를 결정 → 이것이 에이전트 루프의 본질

## 기능 확장 레이어
```
[내장 도구] - 기초
    ├─ Skills     → Claude가 알 수 있는 것 확장
    ├─ MCP        → 외부 서비스 연결
    ├─ Hooks      → 워크플로우 자동화
    └─ Subagents  → 작업 위임
```

# 세션

## 기본 개념
- 대화(메시지, 도구 사용, 결과)를 **로컬에 자동 저장**
- 코드 변경 전 파일 **스냅샷 자동 생성** → 롤백 가능
- 각 세션은 **독립적** (이전 대화 기록 없이 새 컨텍스트로 시작)
- 세션 간 지식 유지: **자동 메모리** 
- 지속적인 지침 추가: **CLAUDE.md**

## 브랜치 간 작업
- 세션은 **현재 디렉토리에 연결**
- 브랜치 전환 시 파일은 새 브랜치 기준, **대화 기록은 유지**
- **git worktrees** 활용 시 개별 브랜치에 대한 별도 디렉토리에서 병렬 세션 실행 가능

# 컨텍스트 윈도우

컨텍스트 윈도우에 포함되는 항목:
- 대화 기록
- 파일 콘텐츠
- 명령 출력
- CLAUDE.md 
- 로드된 Skills
- 시스템 지침

## 컨텍스트가 가득 찰 때
Claude Code는 한계에 접근할 때 컨텍스트를 자동으로 관리한다.

**자동 처리 순서:**
1. 오래된 도구 출력 제거
2. 대화 요약 (초반의 상세 지침 손실 가능)

**대응 방법:**
- 대화 기록에 의존하기보다는 지속적인 규칙은 **CLAUDE.md**에 추가
- 압축 중에 보존되는 것 제어
	- **CLAUDE.md**에 “Compact Instructions” 섹션 추가
	- `/compact`를 포커스와 함께 실행 (예: `/compact focus on the API changes`)
- /context 명령어로 사용량 확인
- /mcp 명령어로 MCP 서버별 비용 확인

## Skills & Subagents로 컨텍스트 최적화
| 기능 | 동작 방식 |
|---|---|
| **Skills** | 세션 시작 시 설명만 로드, **전체 내용은 사용 시점에 로드** |
| **Subagents** | **완전히 분리된 독립 컨텍스트**로 실행, 완료 후 요약만 반환 |

# 체크포인트
모든 파일 편집은 되돌릴 수 있다. 

롤백이 가능한 이유:
- 파일 편집 전 **자동 스냅샷 생성**
- 세션 로컬 저장, **git과 별개**

---

# 특정 명령 자동 허용
- `.claude/settings.json`에서 신뢰할 수 있는 명령을 허용 목록에 추가하면 매번 확인 요청 없이 자동 실행된다.
- 설정 범위: 조직 전체 정책 → 개인 선호도까지 조정 가능

# 팁

- 구현 전에 탐색한다:
	- 복잡한 문제의 경우 분석과 작업을 분리
	- 계획 모드를 사용하여 계획을 검토/개선하고 작업 수행

- 지시하지 말고 위임한다:
	- 능력있는 동료에게 위임하는 것처럼 생각한다.
	- 컨텍스트와 방향을 제공하고 세부 사항을 파악하는 일은 위임한다.
	```
	// 예시
	체크아웃 흐름이 만료된 카드를 가진 사용자에게 손상되었습니다.
	관련 코드는 src/payments/에 있습니다. 조사하고 수정할 수 있나요?
	```
