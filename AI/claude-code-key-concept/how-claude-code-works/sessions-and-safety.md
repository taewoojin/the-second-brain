# 세션과 안전장치

> **TL;DR**
> 각 세션은 독립적이며 대화 내용이 로컬에 저장된다. 세션을 재개하거나 포크할 수 있고, 컨텍스트 윈도우가 차면 자동으로 압축된다. 안전장치로는 파일 변경을 되돌리는 체크포인트와 Claude의 행동 범위를 제어하는 권한 모드가 있다.

## 세션의 기본 동작

Claude Code는 작업하면서 대화를 로컬에 저장한다. 메시지, 도구 사용, 결과가 모두 기록되어 세션을 되감거나 재개하거나 포크할 수 있다. 코드를 변경하기 전에는 해당 파일의 스냅샷도 저장해서 필요하면 되돌릴 수 있다.

**세션은 독립적이다.** 새 세션은 이전 세션의 대화 이력 없이 빈 컨텍스트 윈도우로 시작한다. 세션 간에 정보를 유지하려면 자동 메모리나 CLAUDE.md를 사용한다.

## 브랜치와 세션

세션은 현재 디렉토리에 묶인다. 재개할 때 해당 디렉토리의 세션만 표시된다.

브랜치를 전환하면 Claude는 새 브랜치의 파일을 보지만, 대화 이력은 그대로 유지된다. 브랜치를 바꿔도 이전에 논의한 내용을 기억한다.

병렬로 Claude 세션을 실행하려면 git worktree를 사용하면 된다. worktree는 브랜치마다 별도의 디렉토리를 생성하므로 각각 독립적인 세션을 가질 수 있다.

## 세션 재개와 포크

`claude --continue` 또는 `claude --resume`으로 세션을 재개하면, 같은 세션 ID로 이어서 작업한다. 새 메시지는 기존 대화에 추가된다. 대화 이력은 복원되지만 세션 범위 권한은 복원되지 않으므로 다시 승인해야 한다.

```
세션 A ─── 메시지1 ─── 메시지2 ─── 메시지3
                                     │
                    resume ──────────┤──── 메시지4 (같은 세션 ID)
                                     │
                    fork ────────────┴──── 메시지4 (새 세션 ID)
```

다른 접근을 시도하되 원래 세션은 그대로 두고 싶으면 `--fork-session` 플래그를 사용한다.

```bash
claude --continue --fork-session
```

대화 이력은 그 시점까지 유지하면서 새 세션 ID가 생긴다. 원래 세션은 변경되지 않는다.

**같은 세션을 여러 터미널에서 재개하면:** 두 터미널 모두 같은 세션 파일에 쓴다. 메시지가 뒤섞이며 각 터미널은 자기 메시지만 보이지만, 나중에 재개하면 뒤섞인 전체가 보인다. 같은 시작점에서 병렬 작업을 하려면 `--fork-session`을 사용해 각 터미널에 독립 세션을 주는 것이 좋다.

## 컨텍스트 윈도우

컨텍스트 윈도우에는 대화 이력, 파일 내용, 명령 출력, CLAUDE.md, 자동 메모리, 로드된 스킬, 시스템 지시가 들어간다. 작업하면서 컨텍스트가 채워진다.

### 컨텍스트가 가득 차면

Claude Code가 자동으로 관리한다. 오래된 도구 출력을 먼저 지우고, 필요하면 대화를 요약한다. 사용자 요청과 핵심 코드 조각은 보존되지만, 대화 초반의 상세한 지시는 사라질 수 있다. 지속적으로 유지해야 하는 규칙은 CLAUDE.md에 넣는 것이 좋다.

압축 시 보존할 내용을 제어하려면:
- CLAUDE.md에 "Compact Instructions" 섹션을 추가하거나
- `/compact`에 초점을 지정한다 (예: `/compact focus on the API changes`)

`/context`로 무엇이 컨텍스트를 차지하는지 확인할 수 있다. MCP 서버는 매 요청마다 도구 정의를 추가하므로, 서버 몇 개만으로도 상당한 컨텍스트를 차지할 수 있다. `/mcp`로 서버별 비용을 확인한다.

### 스킬과 서브에이전트를 통한 컨텍스트 관리

**스킬**은 필요할 때만 로드된다. 세션 시작 시에는 스킬 설명만 보이고, 실제 사용할 때 전체 내용이 로드된다. 수동으로 호출하는 스킬은 `disable-model-invocation: true`를 설정하면 설명조차 컨텍스트에 넣지 않을 수 있다.

**서브에이전트**는 자체적인 별도 컨텍스트를 가진다. 메인 대화의 컨텍스트를 소모하지 않으며, 작업이 끝나면 요약만 반환한다. 긴 세션에서 서브에이전트가 유용한 이유다.

## 체크포인트로 변경사항 되돌리기

**모든 파일 편집은 되돌릴 수 있다.** Claude가 파일을 편집하기 전에 현재 내용의 스냅샷을 저장한다. 문제가 생기면 `Esc`를 두 번 눌러 이전 상태로 되감거나, Claude에게 되돌리라고 요청하면 된다.

체크포인트는 세션에 로컬이며 git과 별개다. 파일 변경만 대상이다. 원격 시스템(데이터베이스, API, 배포)에 영향을 주는 행동은 체크포인트할 수 없기 때문에, Claude는 외부 부작용이 있는 명령을 실행하기 전에 확인을 요청한다.

## 권한 제어

`Shift+Tab`으로 권한 모드를 순환할 수 있다.

| 모드 | 동작 |
|------|------|
| **기본(Default)** | 파일 편집과 셸 명령 실행 전에 확인을 요청 |
| **편집 자동 승인(Auto-accept edits)** | 파일 편집은 자동 승인, 명령은 여전히 확인 요청 |
| **플랜 모드(Plan mode)** | 읽기 전용 도구만 사용하여 계획을 세우고, 사용자가 승인한 후 실행 |

`.claude/settings.json`에서 특정 명령(예: `npm test`, `git status`)을 허용해두면 매번 확인하지 않아도 된다. 설정은 조직 전체 정책부터 개인 선호까지 범위를 지정할 수 있다.
