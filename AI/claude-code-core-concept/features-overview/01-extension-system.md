# 확장 시스템 소개

> **한눈에 보기**
> Claude Code는 여러 확장 기능을 제공한다: 항상 작동하는 CLAUDE.md와 Rules, 필요할 때 쓰는 Skills·Subagents·Agent Teams, 외부 서비스를 연결하는 MCP, 이벤트에 반응하는 Hooks. 이것들을 묶어 배포하는 단위가 Plugins다.

## 확장 기능은 왜 필요한가

Claude Code에는 파일 읽기/쓰기, 검색, 명령 실행, 웹 접근 같은 도구가 내장되어 있다. 이것만으로 대부분의 코딩 작업은 가능하다. 하지만 다음과 같은 상황에서는 내장 도구만으로 부족하다:

- 매번 같은 코딩 컨벤션을 알려줘야 할 때 → **CLAUDE.md**
- 배포 절차 같은 반복 워크플로를 실행하고 싶을 때 → **Skills**
- 대량의 파일을 읽되, 메인 대화의 컨텍스트를 오염시키고 싶지 않을 때 → **Subagents**
- 여러 작업을 독립적으로 병렬 수행하고 싶을 때 → **Agent Teams**
- 데이터베이스 조회나 Slack 전송 같은 외부 시스템 연동이 필요할 때 → **MCP**
- 파일 편집 후 린트를 자동으로 돌리고 싶을 때 → **Hooks**

각 기능은 Claude Code의 에이전트 루프(agentic loop) — 컨텍스트 파악 → 작업 수행 → 결과 검증의 반복 — 중 서로 다른 지점에 연결된다.

---

## CLAUDE.md

프로젝트 컨벤션, 빌드 명령어, "항상 이렇게 해라" 같은 규칙을 적어두는 곳이다. 세션이 시작될 때 자동으로 로드되므로, 매번 같은 말을 반복할 필요가 없다.

**적합한 용도:**
- 코딩 컨벤션 (예: "pnpm을 사용하라, npm 금지")
- 빌드/테스트 명령어
- 프로젝트 구조 설명
- "절대 하지 마라" 규칙

**예시:**
```markdown
# CLAUDE.md
- 패키지 매니저는 pnpm을 사용한다
- 커밋 전에 반드시 테스트를 실행한다
- API 응답은 항상 camelCase로 반환한다
```

**권장 사항:** 200줄을 넘으면 참고 자료는 Skills로, 경로별 규칙은 Rules로 분리하기 시작한다. 최대 500줄을 넘지 않도록 한다.

---

## Rules

CLAUDE.md가 길어지면 주제별·경로별로 분리하는 규칙 파일들이다. `.claude/rules/` 디렉토리에 개별 마크다운 파일로 저장한다. 모든 규칙이 매 세션에 로드되거나, frontmatter에 `paths`를 지정하면 해당 파일을 작업할 때만 로드된다.

**적합한 용도:**
- 언어별 가이드라인 (예: `python-style.md`, `typescript-conventions.md`)
- 특정 디렉토리에만 적용되는 규칙

**CLAUDE.md와의 관계:** CLAUDE.md가 커지면 Rules로 분리한다. CLAUDE.md에는 핵심 컨벤션과 빌드 명령어만 남기고, 세부 가이드라인은 Rules에 넣는다.

## Skills

Skills는 Claude가 필요할 때 꺼내 쓰는 지식과 워크플로로, 두 가지 형태가 있다:

- **참고 자료형** — API 스타일 가이드, 스키마 문서 등. Claude가 관련 작업을 할 때 자동으로 참조한다.
- **실행형** — `/deploy`, `/review` 같은 명령으로 호출하는 워크플로. 특정 절차를 단계별로 수행한다.

CLAUDE.md와 달리 매 세션에 전체 내용이 로드되지 않는다. 세션 시작 시에는 이름과 설명만 로드되고, 실제로 사용할 때 전체 내용이 로드된다. 따라서 컨텍스트 비용이 낮다.

**적합한 용도:**
- 배포 체크리스트 (`/deploy`)
- 코드 리뷰 워크플로 (`/review`)
- API 엔드포인트 패턴 문서
- 팀 고유의 작업 절차

**예시 — 실행형 Skill:**
```markdown
---
name: deploy
description: 프로덕션 배포 체크리스트
---
1. 테스트 전체 통과 확인
2. 스테이징 환경에 배포
3. 스모크 테스트 실행
4. 프로덕션 배포
```

## Subagents

Subagent는 자체 컨텍스트 윈도우에서 작동한다. 수십 개의 파일을 읽거나 광범위한 검색을 수행하더라도, 메인 대화에는 요약된 결과만 전달된다. 이로써 메인 대화의 컨텍스트가 오염되지 않는다.

**적합한 용도:**
- 많은 파일을 읽어야 하는 조사 작업
- 병렬로 수행 가능한 독립 작업
- 특정 Skills를 미리 로드한 전문 워커

**작동 방식:**
1. 메인 Claude가 Subagent를 생성하며 작업 지시를 전달
2. Subagent는 독립된 컨텍스트에서 작업 수행
3. 작업 완료 후 결과 요약을 메인 대화에 반환

Subagent는 메인 세션의 대화 기록이나 활성화된 Skills를 상속하지 않는다. 필요한 Skills는 생성 시 명시적으로 지정해야 한다.

---

## Agent Teams

Subagent가 "메인에 보고하는 워커"라면, Agent Teams는 "동료끼리 논의하는 팀"이다. 각 팀원은 독립적인 Claude Code 세션이며, 공유 태스크 리스트를 통해 조율하고 서로에게 직접 메시지를 보낼 수 있다.

**적합한 용도:**
- 보안, 성능, 테스트를 동시에 리뷰하는 병렬 코드 리뷰
- 경쟁 가설을 세우고 검증하는 디버깅
- 각 팀원이 별도 모듈을 담당하는 신규 기능 개발

**Subagent와의 차이:** Subagent는 결과만 돌려주지만, Agent Teams의 팀원들은 서로 발견한 것을 공유하고, 서로의 판단에 이의를 제기할 수 있다.

> Agent Teams는 실험적 기능이며, 기본적으로 비활성화되어 있다.

---

## MCP (Model Context Protocol)

MCP가 없으면 Claude는 로컬 파일 시스템과 웹 검색만 가능하다. MCP 서버를 연결하면 데이터베이스 조회, Slack 메시지 전송, 브라우저 제어 같은 외부 작업이 가능해진다.

**적합한 용도:**
- 데이터베이스 쿼리
- Slack, GitHub 등 외부 서비스와 상호작용
- 브라우저 자동화

**작동 방식:** 세션 시작 시 연결된 MCP 서버의 도구 정의와 스키마가 로드된다. Claude는 이 도구들을 내장 도구처럼 사용한다.

**주의:** MCP 연결은 세션 중간에 끊어질 수 있다. 끊어지면 해당 도구가 경고 없이 사라진다. 문제가 의심되면 `/mcp` 명령으로 상태를 확인한다.

---

## Hooks

Hooks는 Claude의 판단이 개입하지 않는 확정적(deterministic) 자동화다. 파일 편집 후 린트 실행, 세션 시작 시 환경 설정 같은 작업에 적합하다.

**적합한 용도:**
- 파일 편집 후 ESLint 자동 실행
- 세션 시작/종료 시 로깅
- 특정 파일 수정 시 알림 전송

**작동 방식:** 도구 실행, 세션 시작/종료, 프롬프트 제출, 권한 요청, 컨텍스트 압축 등의 이벤트에 반응하여 등록된 스크립트가 실행된다. 기본적으로 컨텍스트 비용은 제로다(스크립트가 출력을 반환하는 경우 제외).

---

## Plugins와 Marketplaces

Plugin은 위에서 설명한 기능들을 패키징하는 계층이다. 같은 설정을 여러 저장소에서 재사용하거나, 다른 사람에게 배포할 때 사용한다.

- Plugin의 Skills는 네임스페이스로 구분된다 (예: `/my-plugin:review`). 여러 Plugin이 공존할 수 있다.
- Marketplace는 Plugin을 호스팅하고 배포하는 플랫폼이다.

---

## 기능 간 계층 구조

같은 기능이 여러 수준(사용자, 프로젝트, 플러그인, 관리 정책)에서 정의될 수 있다. 이때 적용 규칙은 다음과 같다:

- **CLAUDE.md** — 모든 수준의 내용이 합산된다. 충돌 시 Claude가 판단하여 조율하며, 대체로 더 구체적인 지시가 우선한다.
- **Skills, Subagents** — 같은 이름이면 하나만 적용된다. 우선순위: 관리 정책 > 사용자 > 프로젝트.
- **MCP 서버** — 같은 이름이면 하나만 적용된다. 우선순위: 로컬 > 프로젝트 > 사용자.
- **Hooks** — 모든 수준의 Hooks가 합산 실행된다.

## 기능 조합 패턴

각 기능은 단독으로도 유용하지만, 조합하면 더 강력해진다.

| 조합 | 작동 방식 | 예시 |
|------|----------|------|
| **Skill + MCP** | MCP가 연결을 제공하고, Skill이 사용법을 알려줌 | MCP로 DB 연결, Skill에 스키마와 쿼리 패턴 문서화 |
| **Skill + Subagent** | Skill이 Subagent를 생성하여 병렬 작업 수행 | `/audit` Skill이 보안·성능·스타일 Subagent를 각각 실행 |
| **CLAUDE.md + Skills** | CLAUDE.md에 항상 적용할 규칙, Skills에 참고 자료 | CLAUDE.md에 "API 컨벤션을 따르라", Skill에 API 스타일 가이드 전문 |
| **Hook + MCP** | Hook이 MCP를 통해 외부 액션 트리거 | 중요 파일 수정 시 Slack 알림 전송 |
