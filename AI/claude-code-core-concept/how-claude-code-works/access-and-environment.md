# 접근 범위와 실행 환경

> **한눈에 보기**
> `claude`를 실행하면 프로젝트 파일, 터미널, git 상태, CLAUDE.md, 자동 메모리, 설정된 확장 기능에 접근할 수 있다. 실행 환경은 로컬/클라우드/리모트 컨트롤 세 가지이며, 터미널 외에 데스크톱 앱, IDE, 웹 등 다양한 인터페이스로 사용할 수 있다.

## Claude가 접근할 수 있는 것

디렉토리에서 `claude`를 실행하면 다음에 접근할 수 있다.

| 대상 | 설명 |
|------|------|
| **프로젝트** | 현재 디렉토리와 하위 디렉토리의 파일. 권한을 주면 다른 경로의 파일에도 접근 가능 |
| **터미널** | 빌드 도구, git, 패키지 매니저, 시스템 유틸리티, 스크립트 등 커맨드라인에서 실행할 수 있는 모든 것 |
| **git 상태** | 현재 브랜치, 커밋되지 않은 변경사항, 최근 커밋 이력 |
| **CLAUDE.md** | 프로젝트별 지침, 컨벤션, 컨텍스트를 저장하는 마크다운 파일. 매 세션마다 로드됨 |
| **자동 메모리(Auto Memory)** | 작업하면서 Claude가 자동으로 저장하는 학습 내용 — 프로젝트 패턴, 사용자 선호 등. MEMORY.md의 처음 200줄이 세션 시작 시 로드됨 |
| **확장 기능** | MCP 서버, 스킬, 서브에이전트, Claude in Chrome 등 사용자가 설정한 것들 |

Claude는 프로젝트 전체를 볼 수 있기 때문에, "인증 버그를 고쳐줘"라고 요청하면 관련 파일을 검색하고, 여러 파일을 읽어 맥락을 파악하고, 파일 간에 걸친 수정을 하고, 테스트를 돌리고, 커밋까지 할 수 있다. 현재 파일만 보는 인라인 코드 어시스턴트와 다른 점이다.

## 실행 환경

Claude Code는 세 가지 환경에서 실행할 수 있다. 코드가 어디서 실행되는지가 다르다.

| 환경 | 코드 실행 위치 | 용도 |
|------|--------------|------|
| **로컬(Local)** | 사용자의 머신 | 기본값. 파일, 도구, 환경에 완전히 접근 가능 |
| **클라우드(Cloud)** | Anthropic이 관리하는 VM | 작업을 오프로드하거나, 로컬에 없는 저장소에서 작업 |
| **리모트 컨트롤(Remote Control)** | 사용자의 머신 (브라우저에서 제어) | 웹 UI를 사용하면서 모든 것은 로컬에서 실행 |

## 인터페이스

터미널, 데스크톱 앱, IDE 확장(VS Code, JetBrains), claude.ai/code, 리모트 컨트롤, Slack, CI/CD 파이프라인을 통해 Claude Code에 접근할 수 있다. 인터페이스가 달라도 내부의 에이전틱 루프는 동일하다.
