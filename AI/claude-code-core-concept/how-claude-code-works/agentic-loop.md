# 에이전틱 루프(Agentic Loop)

> **한눈에 보기**
> Claude Code는 "컨텍스트 수집 → 행동 → 결과 검증"을 반복하는 에이전틱 루프로 작동한다. 루프를 구성하는 두 축은 추론을 담당하는 **모델**과 실제 행동을 수행하는 **도구**다. Claude Code는 이 둘을 연결하는 에이전틱 하네스(agentic harness) 역할을 한다.

## 3단계 루프

작업을 요청하면 Claude는 세 단계를 반복하며 진행한다.
![에이전틱 루프](agentic-loop.svg)

세 단계가 항상 고르게 실행되는 것은 아니다. 작업 성격에 따라 비중이 달라진다.
- **코드베이스에 대한 질문** — 컨텍스트 수집만으로 끝날 수 있다
- **버그 수정** — 세 단계를 여러 차례 반복한다
- **리팩토링** — 검증 단계에 시간이 많이 걸린다

각 단계에서 얻은 정보가 다음 단계의 판단 근거가 된다. Claude는 수십 개의 행동을 연쇄적으로 실행하면서 중간중간 방향을 수정한다.

사용자도 루프의 일부다. 진행 중 언제든 끼어들어 방향을 바꾸거나, 추가 정보를 제공하거나, 다른 접근을 시도하게 할 수 있다.

## 모델(Model)

모델은 코드를 읽고, 구성 요소 간의 관계를 파악하고, 목표 달성을 위해 무엇을 변경해야 하는지 추론하는 역할을 한다. 복잡한 작업은 단계별로 분해하고, 각 단계의 결과를 바탕으로 다음 행동을 조정한다.

여러 모델을 선택할 수 있으며 각각 장단점이 다르다.

- **Sonnet** — 대부분의 코딩 작업에 적합
- **Opus** — 복잡한 아키텍처 결정처럼 강한 추론이 필요할 때 적합

세션 중 `/model`로 전환하거나, 시작 시 `claude --model <이름>`으로 지정할 수 있다.

> 이 문서에서 "Claude가 선택한다", "Claude가 판단한다"라고 표현하는 것은 모델이 추론을 수행한다는 뜻이다.

## 도구(Tool)

도구가 없으면 Claude는 텍스트로만 응답할 수 있다. 도구가 있어야 비로소 파일을 읽고, 코드를 수정하고, 명령을 실행하는 등의 행동이 가능해진다. 각 도구 사용의 결과가 루프에 다시 피드백되어 다음 판단의 근거가 된다.

### 도구의 5가지 카테고리

| 카테고리 | Claude가 할 수 있는 것 |
|----------|----------------------|
| **파일 작업** | 파일 읽기, 코드 편집, 새 파일 생성, 이름 변경 및 재구성 |
| **검색** | 패턴으로 파일 찾기, 정규식으로 내용 검색, 코드베이스 탐색 |
| **실행** | 셸 명령 실행, 서버 시작, 테스트 실행, git 사용 |
| **웹** | 웹 검색, 문서 가져오기, 에러 메시지 조회 |
| **코드 인텔리전스** | 편집 후 타입 에러/경고 확인, 정의로 이동, 참조 찾기 (코드 인텔리전스 플러그인 필요) |

이 외에도 서브에이전트 생성, 사용자에게 질문하기 등 orchestration 도구가 있다.

### 도구 체이닝 예시

"실패하는 테스트를 고쳐줘"라고 요청하면 Claude는 다음과 같이 도구를 연쇄 사용한다.

1. 테스트 스위트를 실행하여 실패 항목 확인
2. 에러 출력 읽기
3. 관련 소스 파일 검색
4. 해당 파일을 읽어 코드 이해
5. 파일을 편집하여 문제 수정
6. 테스트를 다시 실행하여 수정 검증

각 단계의 결과가 다음 단계를 결정한다. 이것이 에이전틱 루프의 실제 작동 모습이다.

## 확장 기능

기본 제공 도구 위에 추가 기능을 쌓을 수 있다.

| 확장 기능 | 역할 |
|----------|------|
| **스킬(Skill)** | Claude가 알고 있는 것을 확장 — 워크플로우, 패턴, 가이드 등 |
| **MCP** | 외부 서비스와 연결 |
| **훅(Hook)** | 워크플로우 자동화 |
| **서브에이전트(Subagent)** | 작업을 위임하여 병렬 처리 |
