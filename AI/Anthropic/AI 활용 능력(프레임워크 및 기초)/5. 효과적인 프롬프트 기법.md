
## 프롬프팅이란?

프롬프팅은 AI에게 원하는 것, 원하는 방식, 그리고 협업 과정 전반에 걸쳐 어떻게 상호작용하고 싶은지를 명확하게 전달하는 것입니다. 
마치 열정적이지만 명확한 방향 설정이 필요한 새 동료에게 업무를 설명하는 것과 같습니다.

---

## 프롬프트 엔지니어링이란?

AI 시스템이 정확히 무엇을 원하는지 이해할 수 있도록 효과적인 지시를 설계하는 것입니다. 좋은 인간 대화의 원칙(명확성, 관련 맥락 제공, 구체적 예시)이 AI에도 적용되지만, 차이점도 있습니다:
- 인간이 자연스럽게 추론할 수 있는 것도 더 명시적으로 설명해야 함
- AI의 제한된 컨텍스트 윈도우를 고려해야 함
- AI가 처리하기 쉬운 특정 포맷 사용이 필요할 수 있음

## 6가지 핵심 프롬프팅 원칙

### 1. 맥락(Context) 제공하기

단순히 "무엇"을 원하는지뿐 아니라 "왜" 필요한지, "누가" 묻는지까지 전달합니다.

**모호한 질문:**
> "기후 변화에 대해 이야기해주세요"

**개선된 질문:**
> 열대 지역 농업에 대한 기후변화의 3가지 주요 영향을 지난 10년간의 사례와 함께 설명해 주세요.
> 인도네시아 농업연구소 면접 준비 중이고, 생태학 학위는 있지만 기후변화 전문 지식은 없습니다.


### 2. 좋은 예시 보여주기 (Few-shot Prompting)

원하는 출력 스타일을 설명하기 어려울 때, 예시를 제공하면 효과적입니다.
```
기술 용어를 쉬운 말로 바꾸는 예시입니다:
  - 원문: 양자 알고리즘은 제곱에 비례하는 속도 향상을 보입니다
  - 변환: 새로운 방법은 기존보다 문제를 대략 두 배 빠르게 해결합니다

이제 이 설명을 같은 방식으로 바꿔주세요: 
```
다양한 케이스를 커버하는 예시를 제공하면 AI가 패턴을 더 잘 이해합니다.

## 3. 출력 제약조건 명시하기

형식, 길이, 사용할 언어, 디자인 요소 등을 구체적으로 지정합니다.
```
깔끔하고 현대적인 싱글페이지 포트폴리오 웹사이트를 만들어줘.
포함할 섹션: hero, about me, skills, portfolio, experience, contact
내비게이션: sticky, 반응형, 모바일에서는 햄버거 메뉴
색상: sunset 팔레트, 다크/라이트 모드 토글 포함
```

## 4. 복잡한 작업을 단계별로 나누기 (Chain of Thought)

실행 과정을 명시하면 의도한 방식대로 작업이 진행됩니다.
```
분기별 매출 데이터를 다음 순서로 분석해 주세요:
1. 최고 실적 제품 식별
2. 이전 분기와 비교
3. 특이한 트렌드나 패턴 강조
4. 트렌드의 가능한 원인 제안
```
도메인 전문가로서 쌓은 경험과 지식이 작업 실행에 중요할수록, 그 지식을 프롬프트에 담는 것이 좋습니다.

## 5. 먼저 생각하도록 요청하기

작업 실행 "전에" 생각할 공간을 주면 더 깊이 있는 응답을 얻을 수 있습니다.
```
답변하기 전에 이 문제를 신중하게 생각해 보세요.
최적의 해결책을 제시하기 전에 관련된 다양한 요소, 잠재적 제약 조건, 다양한 접근 방식을 고려해 보세요.
```
이렇게 하면 AI가 어디서 잘못된 방향으로 가는지 파악하기도 쉬워집니다.

## 6. 역할, 스타일, 톤 정의하기

AI가 어떤 관점에서 접근하고 어떻게 소통할지 지정합니다.

>경험 많은 과학 교사의 관점에서, 과학에 관심 있는 
>똑똑한 10살 아이에게 무지개가 어떻게 형성되는지 설명해 주세요.

>UX 디자인 전문가로서 이 웹사이트 와이어프레임을 검토하고 
>사용자 내비게이션과 접근성에 초점을 맞춰 3가지 개선점을 제안해 주세요.


##  반복과 실험

효과적인 프롬프팅은 반복적이고 실험적인 과정입니다.

**응답이 기대와 다를 때 시도할 것들:**
- 더 많은 구체성이나 맥락 추가
- 원하는 출력의 예시 제공
- 작업을 더 작은 단계로 분해
- 다른 기법이나 기법의 조합 시도
- 변형 요청: "이것의 3가지 다른 버전을 줄 수 있어?"
- 형식 변경 요청: "문단 대신 인터랙티브 아티팩트로 보여줄 수 있어?"
- 신뢰도 확인: "이 답변에 대해 얼마나 확신해?"
- 대화를 완전히 새로 시작

**일반적인 실수:**
- AI가 마음을 읽을 수 있다고 가정
- 하나의 프롬프트에 관련 없는 여러 작업 과도하게 포함
- 성공의 기준이 무엇인지 모호하게 설명
- 이전 응답에 대한 피드백 미제공

## 마무리
프롬프트 엔지니어링은 끊임없이 발전하는 분야입니다. 
모델이 개선됨에 따라 일부 기술은 덜 필요해질 수 있지만, **좋은 소통 원칙**은 여전히 유효합니다. 
실험 정신을 유지하고 결과에 따라 접근 방식을 조정하세요.
