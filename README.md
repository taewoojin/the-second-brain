# the-second-brain
미래의 내가, 과거의 나에게 질문했을 때 즉시 답을 찾을 수 있는 장소

---

## 아이디어의 출발점

Andrej Karpathy의 [LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)에서 출발했다.

대부분의 LLM 활용 방식은 RAG다. 질문할 때마다 원본 문서에서 관련 청크를 검색해 맥락에 넣고 답을 생성한다. Karpathy는 이 방식의 근본적인 한계를 지적한다:

> "LLM은 매 질문마다 지식을 처음부터 재발견한다. 누적이 없다."

> "지식은 한 번 컴파일되고 이후 최신 상태로 유지된다. 매 쿼리마다 재도출하지 않는다."

> "직접 수집한 원본 문서들... 이것들은 불변이다 — LLM은 읽기만 하고 절대 수정하지 않는다."

> "위키는 본인이 직접 쓰지 않는다(혹은 거의 쓰지 않는다) — LLM이 전부 작성하고 유지한다."

> "위키는 지속적으로 누적되는 산출물이다."

Karpathy가 이 관계를 가장 잘 압축한 문장:

> "Obsidian은 IDE고, LLM은 프로그래머고, 위키는 코드베이스다."

---

## 이 프로젝트에서의 구현

Karpathy의 패턴을 그대로 따른다.

- **`raw/`** — 인간이 수집하는 원본 자료. 불변. LLM은 읽기만 한다.
- **`wiki/`** — LLM이 소유하고 유지하는 컴파일된 지식. 인간은 직접 쓰지 않는다.
- **`CLAUDE.md`** — schema document. 위키의 구조, 규칙, 페이지 형식을 LLM에게 주입한다.

세 가지 operation으로 위키를 운영한다:

- **ingest** — `raw/`의 새 자료를 읽고 `wiki/`에 페이지를 생성하거나 갱신
- **query** — `wiki/`를 참조해 질문에 종합 답변 생성
- **lint** — 깨진 링크, 누락 필드, 페이지 간 모순 탐지

인간의 역할은 자료 수집과 질의다. 정리·교차참조·일관성 유지는 LLM이 담당한다.
