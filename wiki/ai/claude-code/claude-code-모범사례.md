# Claude Code 모범 사례

**출처**: https://code.claude.com/docs/en/best-practices | **날짜**: 2026-04-13

## 요약
Claude Code를 효과적으로 활용하기 위한 핵심 패턴을 정리한다. 가장 중요한 제약은 **컨텍스트 윈도우**: 가득 찰수록 성능이 저하되므로 컨텍스트 관리가 최우선이다. 검증 기준 제공, 탐색 우선 계획, 구체적 컨텍스트 제공이 세 가지 핵심 원칙이다.

## 1. 검증 기준을 제공하라

Claude가 스스로 작업을 확인할 수 있을 때 성능이 극적으로 향상된다.

| 전략 | 나쁜 예 | 좋은 예 |
| --- | --- | --- |
| 검증 기준 제공 | "이메일 검증 함수 구현해줘" | "validateEmail 함수 작성. [user@example.com](true), invalid(false), [user@.com](false) 테스트 케이스. 구현 후 테스트 실행해줘" |
| UI 변경 시각적 검증 | "대시보드 더 좋아보이게 만들어줘" | "[스크린샷 붙여넣기] 이 디자인 구현하고 결과 스크린샷과 비교해서 차이 목록 만들고 수정해줘" |
| 근본 원인 해결 | "빌드가 실패해" | "빌드가 이 에러로 실패해: [에러 붙여넣기]. 수정하고 빌드 성공 확인해줘. 에러 억제 말고 근본 원인 해결해줘" |

## 2. 탐색 → 계획 → 코딩 순서로

계획 없이 바로 코딩하면 잘못된 문제를 풀 수 있다. Plan Mode로 구현과 탐색을 분리한다.

**Plan Mode 사용 권장 상황**:
- 변경이 여러 파일에 걸침
- 접근 방식이 불확실함
- 수정하려는 코드에 익숙하지 않음

**Plan Mode 불필요 상황**: 범위가 명확하고 변경이 작을 때 (오타 수정, 로그 추가, 변수 이름 변경 등)

## 3. 구체적인 컨텍스트를 제공하라

| 전략 | 나쁜 예 | 좋은 예 |
| --- | --- | --- |
| 범위 명시 | "foo.py 테스트 추가해줘" | "로그아웃 상태 엣지 케이스 다루는 foo.py 테스트 작성해줘. mock 없이" |
| 출처 지정 | "ExecutionFactory API 왜 이렇게 이상해?" | "ExecutionFactory git 히스토리 보고 API가 어떻게 만들어졌는지 정리해줘" |
| 기존 패턴 참조 | "캘린더 위젯 추가해줘" | "홈페이지 위젯 구현 방식 먼저 봐줘. HotDogWidget.php가 좋은 예시야. 그 패턴대로 월 선택 + 연도 앞뒤 이동 가능한 캘린더 위젯 만들어줘" |
| 증상 설명 | "로그인 버그 고쳐줘" | "세션 만료 후 로그인 실패. src/auth/ 인증 플로우 확인해줘. 토큰 갱신 쪽 특히. 실패 재현하는 테스트 작성 후 수정해줘" |

### 풍부한 컨텍스트 제공 방법
- `@파일명`으로 파일 직접 참조
- 스크린샷/이미지 직접 붙여넣기
- URL로 문서/API 참조 제공
- `cat error.log | claude`로 파이프 입력
- Claude에게 직접 컨텍스트 수집하게 하기

## 4. 환경 설정 투자

### 효과적인 CLAUDE.md 작성
- `/init`으로 프로젝트 기반 스타터 파일 자동 생성 후 점진적 개선
- Claude가 같은 실수를 두 번 하면 → CLAUDE.md에 추가
- 항상 실행하는 명령(빌드, 테스트, 포맷) 명시
- 200줄 이하 유지. 특정 파일 타입용 규칙은 `.claude/rules/`로 분리

### Hooks로 자동화
- 파일 편집 후 자동 포맷팅
- 커밋 전 린트 실행
- Claude 유휴 시 알림

## 5. 컨텍스트 관리 (가장 중요)

컨텍스트 윈도우 관련 팁:
- `/context`로 현재 컨텍스트 사용량 확인
- 사이드 태스크는 Subagent로 위임 (컨텍스트 격리)
- 수동만 사용하는 Skill에 `disable-model-invocation: true` 설정
- `/compact`로 수동 컨텍스트 압축
- 새 주제나 태스크는 새 세션 시작

## 병렬 세션

동일한 작업에서 서로 다른 접근 방식을 탐색할 때:
- `git worktrees`로 별도 디렉토리 생성 → 병렬 Claude 세션 실행 가능
- 각 세션은 독립적 컨텍스트
