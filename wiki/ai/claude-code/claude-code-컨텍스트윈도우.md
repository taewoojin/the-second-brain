# Claude Code 컨텍스트 윈도우 — 작동 원리와 구성

**출처**: `raw/claude-code-context-window.md`
**날짜**: 2026-04-21
**keywords**: context window, 가시성(visibility), 이벤트 종류, 세션 시작 자동로드, subagent 컨텍스트 격리, /compact, 컨텍스트 압축

## 요약

Claude Code의 context window(최대 200,000 토큰)는 시스템 프롬프트·CLAUDE.md·auto memory·MCP 도구 등이 세션 시작 전에 자동 로드되고, 이후 사용자 프롬프트·파일 읽기·훅 실행이 순차적으로 쌓이는 구조다. 각 항목의 가시성 수준(완전 표시·한 줄·숨김)과 /compact 이후 생존 규칙을 이해하는 것이 효율적인 세션 관리의 핵심이다.

## 기본 개념: 가시성(Visibility)

컨텍스트에 존재한다고 해서 터미널에 보이는 것은 아니다. 모든 항목은 세 가지 수준 중 하나를 갖는다.

| 가시성 | 의미 |
|--------|------|
| **완전 표시** (full) | 실제 내용이 터미널에 표시됨 |
| **한 줄 표시** (brief) | 짧은 요약만 표시됨 (예: "Read auth.ts") |
| **숨김** (hidden) | 터미널에 전혀 나타나지 않음 |

## 이벤트 종류(Kind)

| 종류 | 의미 |
|------|------|
| **auto** | 자동으로 로드되는 항목 |
| **you** | 사용자가 직접 입력한 내용 |
| **claude** | Claude의 작업 (파일 읽기, 분석, 수정 등) |
| **hook** | 자동으로 실행되는 훅 |
| **compact** | /compact 명령으로 생성된 요약 |
| **subagent** | 서브에이전트의 별도 컨텍스트 윈도우에서의 작업 |

## 1. 세션 시작 전: 자동 로드 항목

`claude` 실행 시 아무것도 입력하기 전에 아래 항목들이 자동 로드된다.

### 시스템 프롬프트
- **가시성**: 숨김. 항상 가장 먼저 로드됨. 사용자는 절대 볼 수 없음

### 자동 메모리 (Auto memory / MEMORY.md)
- **가시성**: 숨김
- Claude가 이전 세션에서 스스로 기록한 메모
- 앞에서부터 200줄 또는 25KB 중 더 작은 크기까지만 로드됨

### 환경 정보
- **가시성**: 숨김
- 작업 디렉토리, 플랫폼, 쉘, OS 버전, Git 여부 등
- Git 브랜치·상태·최근 커밋은 시스템 프롬프트 맨 끝에 별도 블록으로 로드됨

### MCP 도구 (deferred)
- **가시성**: 숨김
- 사용 가능한 MCP 도구의 **이름만** 먼저 로드됨 (전체 스키마는 필요할 때 개별 로드)
- `ENABLE_TOOL_SEARCH=auto`: 모든 스키마가 컨텍스트의 10% 이내에 맞으면 시작 시 전체 로드
- `ENABLE_TOOL_SEARCH=false`: 시작 시 모든 스키마를 로드 (deferred 없이)

### 스킬 설명
- **가시성**: 숨김
- Claude가 호출할 수 있는 스킬들의 한 줄 설명 목록
- 전체 스킬 내용은 실제로 사용될 때만 로드됨
- `disable-model-invocation: true` 스킬은 이 목록에 없음 — 사용자가 `/스킬명`으로 호출하는 순간까지 컨텍스트에 아예 없음
- `/compact` 이후 재주입되지 않음 — 실제로 호출했던 스킬만 보존됨

### CLAUDE.md 파일들
- **가시성**: 숨김
- 전역(`~/.claude/CLAUDE.md`) 및 프로젝트(`./CLAUDE.md`) 파일이 세션 시작 시 로드됨
- 권장: 200줄 이하로 유지

> **핵심**: 사용자가 아무것도 입력하기 전에 CLAUDE.md, 메모리, 스킬 목록, MCP 도구가 이미 컨텍스트에 있다.

## 2. 사용자 프롬프트

- **가시성**: 완전 표시
- 이미 로드된 항목들에 비하면 상대적으로 매우 작은 크기

## 3. Claude의 작업

### 파일 읽기
- **가시성**: 한 줄 표시 (터미널에 "Read auth.ts"로만 표시됨)
- 파일 내용은 Claude만 봄
- **파일 읽기가 컨텍스트 사용의 주요 원인**
- 팁: 프롬프트를 구체적으로 작성할수록 Claude가 읽는 파일이 줄어듦

### 경로 한정 규칙 (Path-scoped rules)
- **가시성**: 한 줄 표시
- `.claude/rules/`의 `paths:` 패턴 설정 파일은 해당 파일을 읽을 때 자동 로드됨

### grep 및 검색
- **가시성**: 한 줄 표시 (결과는 컨텍스트에 추가되지만 사용자에게는 실행 사실만 보임)

### Claude의 분석과 파일 수정
- **가시성**: 완전 표시 (분석 텍스트, 파일 수정 diff가 터미널에 표시됨)

### 테스트 실행 결과
- **가시성**: 한 줄 표시 (전체 출력은 Claude의 컨텍스트에만 존재)

## 4. 훅 (Hooks)

- **가시성**: 숨김
- `settings.json`에 정의된 훅이 도구 이벤트 후 자동 실행됨

| 훅 출력 방식 | Claude에게 전달 여부 |
|-------------|---------------------|
| `additionalContext` 필드가 있는 JSON 출력 | Claude의 컨텍스트에 전달됨 |
| 종료 코드 2 + stderr | Claude에게 오류로 표면화됨 |
| 종료 코드 0의 일반 stdout | 디버그 로그에만 기록, 컨텍스트에 들어가지 않음 |

> **핵심**: Claude에게 훅 정보를 전달하려면 반드시 `additionalContext` JSON 형식을 사용해야 한다.

## 5. 서브에이전트 (Subagent)

서브에이전트는 **별도의 컨텍스트 윈도우**에서 작동하는 독립된 Claude 인스턴스다.

- 서브에이전트가 읽은 파일들은 **메인 컨텍스트에 들어오지 않음**
- 작업 완료 후 **최종 텍스트 응답 + 작은 메타데이터(토큰 수, 소요 시간)만** 메인 컨텍스트로 돌아옴

### 서브에이전트의 별도 컨텍스트 구성

1. **자체 시스템 프롬프트**: 메인보다 짧음. 메인 세션의 auto memory는 포함되지 않음
2. **프로젝트 CLAUDE.md**: 동일 파일이지만 서브에이전트 컨텍스트에서 소비됨
3. **MCP 도구 + 스킬**: 부모와 동일한 접근 가능 (단, 재귀 방지용 Agent 도구 자체는 기본 제외)
4. **부모가 작성한 태스크 프롬프트**: 사용자 프롬프트 대신 Claude가 작성한 작업 지시를 받음

> **핵심**: 서브에이전트가 대량의 파일을 읽어도 메인 컨텍스트에는 최종 요약만 돌아온다. 이것이 컨텍스트 절약의 핵심 메커니즘이다.

## 6. 특수 입력 방식

### 뱅 명령어 (`!` 접두사)
- **가시성**: 완전 표시
- `!git status`처럼 `!`를 붙여 쉘 명령어를 직접 실행
- 명령어와 출력이 **사용자 메시지의 일부로** 컨텍스트에 추가됨

### `disable-model-invocation: true` 스킬
- 스킬 설명 목록에 없으므로 시작 시 컨텍스트 비용이 0
- 사용자가 `/스킬명`으로 호출하는 순간 전체 스킬 내용이 로드됨
- 권장 사용 사례: 커밋, 배포, 메시지 전송 등 부작용(side effect)이 있는 스킬

## 7. /compact — 컨텍스트 압축

컨텍스트가 많이 쌓여 성능에 영향을 줄 때 사용하는 명령어.

- **가시성**: 한 줄 표시 ("Conversation compacted" 메시지만 보임)
- 대화 내용을 구조화된 요약으로 교체

### 요약에 보존되는 내용
- 사용자의 요청과 의도, 핵심 기술 개념
- 검토하거나 수정한 파일과 중요 코드 스니펫
- 오류와 수정 방법, 미완료 작업

### 요약 후 손실되는 내용
- 도구 출력 원본 전체 (파일 내용 그대로, 테스트 전체 출력 등)
- 중간 추론 과정

## 8. /compact 이후 생존 메커니즘

| 메커니즘 | /compact 이후 상태 |
|----------|-------------------|
| 시스템 프롬프트 | 변화 없음 |
| 프로젝트 루트 CLAUDE.md 및 스코프 없는 규칙 | 디스크에서 자동 재주입됨 |
| 자동 메모리 (Auto memory) | 디스크에서 자동 재주입됨 |
| `paths:` 프론트매터가 있는 경로 한정 규칙 | **소실** — 다음에 매칭 파일을 읽을 때 다시 로드됨 |
| 서브디렉토리의 중첩 CLAUDE.md | **소실** — 해당 서브디렉토리 파일을 읽을 때 다시 로드됨 |
| 호출한 스킬 본문 | 재주입됨 (스킬당 최대 5,000 토큰, 전체 최대 25,000 토큰; 초과 시 오래된 것부터 삭제) |
| 스킬 설명 목록 | **재주입되지 않음** — 실제로 호출했던 스킬만 보존됨 |
| 훅 | 해당 없음 — 컨텍스트와 무관 |

**스킬 본문 주의사항**: 가장 중요한 지시사항을 SKILL.md 상단에 배치할 것. 오래된 스킬이 먼저 삭제된다.

**경로 한정 규칙 주의사항**: 컴팩션 이후에도 규칙을 유지하려면 `paths:` 프론트매터를 제거하거나 프로젝트 루트 CLAUDE.md로 이동할 것.

## 9. 컨텍스트 절약 팁

| 상황 | 팁 |
|------|-----|
| 파일 읽기 | 프롬프트를 구체적으로 작성할수록 Claude가 읽는 파일이 줄어듦 |
| 리서치 작업 | 서브에이전트를 활용하면 대량 파일 읽기가 메인 컨텍스트를 소모하지 않음 |
| 부작용 스킬 | `disable-model-invocation: true`로 설정하면 호출 전까지 컨텍스트 비용이 0 |
| CLAUDE.md 관리 | 200줄 이하로 유지; 참조용 내용은 스킬이나 경로 한정 규칙으로 분리 |
| 훅 출력 | `additionalContext` JSON을 간결하게 작성할 것 |
| 긴 세션 | `/compact`를 사용해 정리; `/context`로 현재 사용량 확인 |
