# Claude Code 작동 원리: 에이전틱 루프

**출처**: https://code.claude.com/docs/en/how-claude-code-works | **날짜**: 2026-04-13

## 요약
Claude Code는 터미널에서 실행되는 에이전틱 어시스턴트로, 세 단계(컨텍스트 수집 → 액션 → 결과 검증)로 구성된 에이전틱 루프를 통해 작동한다. LLM 모델이 추론하고 도구(tool)가 실제 동작을 수행하며, Claude Code는 이 둘을 묶는 에이전틱 하네스(agentic harness) 역할을 한다.

## 에이전틱 루프

사용자가 태스크를 주면 Claude는 다음 세 단계를 반복한다:

1. **컨텍스트 수집** — 파일 읽기, 코드 탐색, 패턴 이해
2. **액션** — 파일 편집, 명령 실행, 코드 작성
3. **결과 검증** — 테스트 실행, 출력 확인, 재시도

단계들은 뚜렷하게 나뉘지 않고 유동적으로 혼합된다. 루프는 태스크가 완료될 때까지 반복되며, 사용자가 언제든 개입해 방향을 전환할 수 있다.

## 도구(Tools) 카테고리

| 카테고리 | 할 수 있는 것 |
| --- | --- |
| **파일 조작** | 파일 읽기, 코드 편집, 파일 생성, 이름 변경 |
| **검색** | 파일 패턴 찾기, 정규식 콘텐츠 검색, 코드베이스 탐색 |
| **실행** | 셸 명령, 서버 시작, 테스트 실행, git |
| **웹** | 웹 검색, 문서 가져오기, 에러 메시지 조회 |
| **코드 인텔리전스** | 타입 에러, 정의 이동, 참조 찾기 (플러그인 필요) |

## 세션과 컨텍스트 윈도우

- 각 세션은 독립적인 컨텍스트 윈도우로 시작 (이전 대화 히스토리 없음)
- `claude --continue` 또는 `claude --resume`으로 이전 세션 재개 가능
- `--fork-session` 플래그로 현재 대화를 분기하여 별도 세션 생성
- **Auto memory**: Claude가 세션 간에 학습 내용을 자동으로 저장 (`~/.claude/projects/`)
- 컨텍스트 윈도우가 가득 차면 자동 compaction 발생 (초기 지시 내용이 사라질 수 있으므로 중요 규칙은 CLAUDE.md에 저장)

## 실행 환경

| 환경 | 코드 실행 위치 | 사용 사례 |
| --- | --- | --- |
| **로컬** | 사용자 머신 | 기본값. 파일/도구/환경 전체 접근 |
| **클라우드** | Anthropic 관리 VM | 태스크 오프로드, 로컬에 없는 저장소 작업 |
| **리모트 컨트롤** | 사용자 머신 (브라우저 제어) | 웹 UI 사용하면서 로컬 유지 |
