# Claude Code 작동 방식: 에이전트 하네스와 내장 도구

**출처**: `raw/claude code의 작동 방식.md` | https://code.claude.com/docs/ko/how-claude-code-works
**날짜**: 2026-04-20
**keywords**: agentic loop, 에이전트 하네스, context window, 세션 관리, checkpoint, 권한 모드, git worktrees

## 요약

Claude Code는 언어 모델을 코딩 에이전트로 변환하는 에이전트 하네스(agent harness)로, 컨텍스트 수집 → 작업 수행 → 결과 검증의 3단계 agentic loop를 반복한다. 파일작업·검색·실행·웹·코드인텔리전스 5가지 도구 범주를 통해 프로젝트 전체에 걸쳐 자율적으로 작동하며, 세션은 독립적이고 체크포인트와 권한 모드로 안전을 보장한다.

## 에이전트 루프 (Agentic Loop)

작업을 받으면 세 단계를 반복한다:

1. **컨텍스트 수집** — 파일 검색, 코드베이스 이해
2. **작업 수행** — 파일 편집, 명령 실행
3. **결과 검증** — 테스트 실행, 변경 확인

루프는 작업 유형에 따라 조정된다. 코드베이스 질문은 컨텍스트 수집만 필요하고, 버그 수정은 세 단계를 반복한다. 사용자는 언제든지 중단하여 방향을 바꾸거나 추가 컨텍스트를 제공할 수 있다.

## 에이전트 하네스 구조

Claude Code = **추론하는 모델** + **작용하는 도구**

> "Claude Code는 Claude 주변의 에이전트 하네스 역할을 합니다: 언어 모델을 능력 있는 코딩 에이전트로 변환하는 도구, 컨텍스트 관리, 실행 환경을 제공합니다."

### 모델

- 모든 언어의 코드를 읽고 구성 요소 간 연결을 이해
- 복잡한 작업은 단계로 나누고 실행하며 학습 내용으로 조정
- Sonnet(대부분의 코딩 작업), Opus(복잡한 아키텍처 결정)
- 세션 중 `/model`로 전환 또는 `claude --model <name>`으로 시작

### 내장 도구

| 범주 | 할 수 있는 것 |
| --- | --- |
| **파일 작업** | 파일 읽기, 코드 편집, 새 파일 생성, 이름 변경 및 재구성 |
| **검색** | 패턴으로 파일 찾기, 정규식으로 콘텐츠 검색, 코드베이스 탐색 |
| **실행** | 셸 명령 실행, 서버 시작, 테스트 실행, git 사용 |
| **웹** | 웹 검색, 문서 가져오기, 오류 메시지 조회 |
| **코드 인텔리전스** | 편집 후 타입 오류 확인, 정의로 이동, 참조 찾기 (플러그인 필요) |

**도구 사용 흐름 예시** ("실패한 테스트 수정해"):
1. 테스트 스위트 실행 → 실패 항목 확인
2. 오류 출력 읽기
3. 관련 소스 파일 검색
4. 파일 읽기 → 코드 이해
5. 파일 편집 → 문제 수정
6. 테스트 재실행 → 검증

## Claude가 접근할 수 있는 것

`claude`를 실행한 디렉토리에서 다음에 접근 가능:

- **프로젝트**: 디렉토리 및 하위 디렉토리의 파일
- **터미널**: 빌드 도구, git, 패키지 관리자, 시스템 유틸리티
- **git 상태**: 현재 브랜치, 미커밋 변경사항, 최근 커밋 기록
- **CLAUDE.md**: 프로젝트별 지침, 규칙, 세션마다 로드되는 컨텍스트
- **Auto Memory**: 세션 간 학습(MEMORY.md 처음 200줄 또는 25KB)
- **설정된 확장**: MCP 서버, skills, subagents, Claude in Chrome

## 세션 관리

세션은 독립적이다. 각 새 세션은 이전 대화 기록 없이 새로운 컨텍스트 윈도우로 시작한다.

### 세션 재개 및 포크

```shell
# 세션 재개 (이전 대화 이어서)
claude --continue
claude --resume

# 세션 포크 (새 ID로 새 브랜치, 원본 유지)
claude --continue --fork-session
```

- 재개 시 대화 기록은 복원되지만 **세션 범위 권한은 복원되지 않음** (다시 승인 필요)
- **여러 터미널에서 동일 세션 재개 시**: 두 터미널의 메시지가 같은 파일에 인터리브됨. 아무것도 손상되지 않지만 대화가 뒤섞임. 나중에 해당 세션을 재개하면 모든 것이 인터리브된 상태로 보임. 병렬 작업에는 `--fork-session`으로 각 터미널에 독립 세션을 부여

### 브랜치 작업

각 Claude Code 대화는 현재 디렉토리에 연결된 세션이다. 브랜치 전환 시 Claude는 새 브랜치 파일을 보지만 대화 기록은 유지된다.

**git worktrees**를 사용하면 개별 브랜치에 대한 별도 디렉토리를 생성하여 병렬 Claude 세션을 실행할 수 있다 → [[wiki/concepts/git-worktrees]] 참조.

### 컨텍스트 윈도우 관리

컨텍스트가 한계에 접근하면 Claude가 자동 관리:
1. 이전 도구 출력 제거
2. 대화 요약

대화 초반 지침은 손실될 수 있으므로 지속적인 규칙은 CLAUDE.md에 기록한다.
- `/context`로 현재 사용량 확인
- `/compact focus on <내용>`으로 포커스 지정 압축 가능
- CLAUDE.md에 "Compact Instructions" 섹션을 추가하면 압축 중 보존되는 것을 제어할 수 있음
- `/mcp`를 실행하여 서버별 컨텍스트 비용 확인 가능

**Skills와 Subagents로 컨텍스트 관리:**
- Skills의 경우 `disable-model-invocation: true`를 설정하면 수동 호출 전까지 설명이 컨텍스트에 로드되지 않아 비용 0
- Subagents는 주 대화와 완전히 분리된 새로운 컨텍스트 윈도우를 사용하므로 작업이 컨텍스트를 부풀리지 않음

## 안전 메커니즘

### 체크포인트 (Checkpoint)

- Claude가 파일을 편집하기 전에 현재 콘텐츠를 스냅샷
- `Esc` 두 번으로 이전 상태 복원 또는 Claude에게 취소 요청
- **세션에 로컬**이며 git과 분리됨. 파일 변경만 다룸
- 원격 시스템(데이터베이스, API, 배포)은 체크포인트 불가

### 권한 모드

`Shift+Tab`으로 순환:

| 모드 | 동작 |
| --- | --- |
| **기본값 (default)** | 파일 편집 및 셸 명령 전에 요청 |
| **자동 수락 편집 (acceptEdits)** | 파일 편집은 자동, 명령은 요청 |
| **계획 모드 (plan)** | 읽기 전용 도구만 사용하여 계획 생성 후 승인 |
| **자동 모드 (auto)** | 백그라운드 안전 검사로 모든 작업 평가 (연구 미리보기) |

`.claude/settings.json`에서 특정 명령(예: `npm test`, `git status`)을 허용하여 매번 요청 없이 실행 가능.

## 효과적인 사용 팁

- **대화형 접근**: 완벽한 프롬프트 불필요. 시작 후 반복적으로 개선
- **구체적인 초기 프롬프트**: 파일명, 제약 조건, 예제 패턴 명시 → 첫 시도 성공률 높아짐
- **검증 기준 제공**: 테스트 케이스, UI 스크린샷, 예상 출력 포함
- **탐색 후 구현**: Plan Mode(`Shift+Tab` 두 번)로 먼저 분석 후 구현
- **위임 방식**: 읽을 파일이나 실행할 명령을 직접 지정하지 말고 목표와 컨텍스트를 제공
