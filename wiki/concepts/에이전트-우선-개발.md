# 에이전트 우선 개발 (Agent-First Development)

**출처**: raw/하네스 엔지니어링 에이전트 우선 세계에서 Codex 활용하기.md | **날짜**: 2026-04-14

## 요약
에이전트 우선 개발은 소프트웨어 엔지니어의 주된 업무가 코드 작성에서 "에이전트가 안정적으로 일할 수 있는 환경, 피드백 루프, 제어 시스템 설계"로 전환되는 개발 패러다임이다. 에이전트가 코드를 생성하고, 사람은 환경을 조율한다.

## 등장 맥락

OpenAI 하네스 팀이 5개월간 수동 코드 0줄로 100만 라인의 소프트웨어 제품을 출시하면서 구체화된 개념. Claude Code, Codex 같은 agentic coding assistant가 장시간 자율적으로 태스크를 수행할 수 있게 되면서 현실화되었다.

## 핵심 원칙

### 1. 에이전트 가독성 최적화
에이전트가 볼 수 없는 것은 존재하지 않는 것. 저장소 외부의 지식(Slack, 구글 문서, 암묵적 지식)은 에이전트에게 없는 것이나 마찬가지.

→ 모든 의사결정, 아키텍처 패턴, 팀 규범을 버전 관리되는 마크다운으로 저장

### 2. 지시 파일은 목차(map), 백과사전(encyclopedia) 아님
거대 단일 지시 파일의 문제:
- 컨텍스트 낭비
- "모든 것이 중요" = 아무것도 중요하지 않음
- 낡은 규칙이 쌓여 신뢰 불가

→ 짧은 진입 파일 + 구조화된 `docs/` 디렉토리로 점진적 공개

### 3. 불변 조건을 기계적으로 강제
문서화만으로는 일관성 유지 불가. 린터, CI, 구조 테스트로 아키텍처 규칙을 자동 강제. 린트 에러 메시지에 수정 지침 포함.

### 4. 엔트로피 관리 자동화
에이전트가 기존 패턴(불균일하더라도) 복제 → 시간이 지나면 드리프트 불가피. 백그라운드 정리 에이전트를 정기적으로 실행해 기술 부채를 조금씩 상환.

## 사람의 새로운 역할

| 기존 역할 | 에이전트 우선 역할 |
| --- | --- |
| 코드 작성 | 에이전트 환경 설계 |
| 코드 리뷰 | 수용 기준 정의 |
| 디버깅 | 에이전트 막힘 = 환경 개선 신호 |
| 문서 작성 | 에이전트가 접근 가능한 형태로 지식 인코딩 |
